## 按模块的重大变化

### 1. 工作流引擎（变化最大）

- **事件驱动工作流 (v1.10.0)**：工作流可以通过事件触发
- **Human-in-the-Loop (v1.13.0)**：工作流可在关键节点暂停，等人工审批/输入
- **HITL 表单增强 (v1.15.0)**：审批表单支持下拉选择、文件上传
- **协作编辑 (v1.14.0)**：多人同时编辑同一个工作流，实时同步
- **支持慢速/长时间运行模型 (v1.15.0)**：适配生图等耗时长的模型

### 2. 知识库 / RAG

- **Summary Index (v1.12.0)**：LLM 给文档片段生成摘要再做向量检索
- **知识库升级 (v1.11.0)**：从单路检索升级为更丰富的索引模式
- **多模态支持**：图文混合摘要（GPT-4V / Claude 3）
- **Hologres (v1.13.1)**：新增 Hologres 作为向量存储后端

### 3. Agent / Skills 体系

- **Agent × Skills (v1.14.0-rc1)**：沙箱化运行时 + Skill 编辑器 + @ 语法调用工具
- **CoT 思考链可视化 (v1.15.0)**：Chatflow 中实时展示模型推理过程
- **动态变量装配**：Pull a Variable 机制

### 4. CLI 工具 (v1.15.0)

- **difyctl**：命令行客户端，终端直接跑 Dify 应用
- 支持 macOS/Linux/Windows，无需 token
- 适合 CI/脚本/Agent 调用

### 5. 基础设施

- MySQL 支持 (v1.10.1)
- MCP 协议支持 (v1.14.0+)
- 插件体系持续增强（Tool OAuth、自动升级）
- 安全加固（多个版本持续修复）

### 6. UI/UX

- 首页和 onboarding 重新设计 (v1.15.0)
- Go-to-anything 全局搜索
- 工作流编辑器可折叠面板
- 无障碍优化

---

## 跟 RAG 评测相关的改进

1. Summary Index (v1.12.0)：摘要索引改变检索质量，评测需关注
2. 知识库稳定性修复 (v1.14.2)：hit-testing 渲染修复
3. HITL (v1.13.0+)：人机协作判断场景
4. difyctl (v1.15.0)：CLI 批量跑评测，自动化更容易

---

## 总结

Dify 这一年从"LLM应用搭建平台"进化成了"AI工作流 + Agent + 协作"的全栈平台。重心排序：工作流能力（HITL/事件驱动/协作）> Agent（Skills/沙箱）> RAG（摘要索引）> 基础设施（MySQL/MCP）。
